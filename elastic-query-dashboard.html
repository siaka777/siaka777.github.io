Elastic Query Languages

Master the four query languages used across the Elastic Stack. Each serves a different purpose — choose a language below to learn and practice.

KQL
Kibana Query Language
The default search language in Kibana's search bars. Designed for simple, human-readable filtering of Elasticsearch data. Used in Discover, dashboards, and alerts.
Kibana · Discover · Alerts
EQL
Event Query Language
Purpose-built for threat hunting and security analytics. Detects sequences of events, relationships between logs, and attack patterns across time-ordered data.
Security · Threat Hunting · SIEM
Lucene
Apache Lucene Query Syntax
The original Elasticsearch query language. Powerful wildcard, fuzzy, range, and proximity searches. Available in Kibana's "switch to Lucene" mode.
Advanced · Wildcards · Fuzzy
ES|QL
Elasticsearch Query Language
A pipe-based query language introduced in Elasticsearch 8.11. Combines search, aggregation, and transformation in a SQL-like pipeline for powerful analytics.
ES 8.11+ · Pipelines · Analytics

ES|QL

Elasticsearch Query Language

ES|QL is a pipe-based query language for analytics and data transformation, introduced in Elasticsearch 8.11. It chains commands with | to process and analyze data.

0/15

// ES|QL Syntax Reference

FROM sourceFROM logs-*Start a query from an index or data stream
WHERE filterFROM logs-* | WHERE status == 200Filter rows (== not :)
STATS aggregation... | STATS count = COUNT(*) BY statusAggregate data by group
SORT... | SORT @timestamp DESCSort results by a field
LIMIT... | LIMIT 100Return only N rows
EVAL computed fields... | EVAL kb = bytes / 1024Create new columns from expressions
KEEP / DROP... | KEEP @timestamp, status, bytesSelect or remove columns from output
RENAME... | RENAME http.status AS statusRename a column
LIKE / RLIKEWHERE message LIKE "error*"LIKE for wildcards, RLIKE for regex
Chained pipelineFROM logs-* | WHERE status >= 400 | STATS n = COUNT(*) BY status | SORT n DESCChain multiple commands together
