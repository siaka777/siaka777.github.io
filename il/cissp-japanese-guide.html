01
Security and Risk Management

セキュリティとリスク管理りすくかんり

15% — 試験出題比率

このドメインは、情報セキュリティじょうほうせきゅりてぃの根幹をなす概念がいねんを扱います。機密性きみつせい・完全性かんぜんせい・可用性かようせい（CIAトライアドとらいあど）を中心に、組織そしきのリスク管理りすくかんり、法的遵守ほうてきじゅんしゅ、セキュリティ方針ほうしんの策定さくていを学びます。CISSP試験の最大出題範囲であり、倫理観りんりかんと職業倫理しょくぎょうりんりも含まれます。

CIA
CIAトライアド：情報セキュリティじょうほうせきゅりてぃの三原則さんげんそく

機密性きみつせい（Confidentiality）：権限けんげんのないものへの情報漏洩じょうほうろうえいを防ぐ。暗号化あんごうか・アクセス制御あくせすせいぎょが主な対策たいさく。

完全性かんぜんせい（Integrity）：データが不正ふせいに改ざんかいざんされていないことを保証ほしょうする。ハッシュ・デジタル署名でじたるしょめいを使用。

可用性かようせい（Availability）：正当せいとうな利用者りようしゃが必要なときに情報じょうほうにアクセスできる状態を維持する。

  • 真正性しんせいせい（Authenticity）：通信の相手が本物であること
  • 否認防止ひにんぼうし（Non-repudiation）：行為を後から否定できない仕組み
  • 説明責任せつめいせきにん（Accountability）：行動が追跡・記録される
KEY TERMS
機密性きみつせい 完全性かんぜんせい 可用性かようせい 否認防止ひにんぼうし
RM
リスク管理りすくかんりの基礎きそプロセス

リスクりすくは「脅威きょうい（Threat）× 脆弱性ぜいじゃくせい（Vulnerability）× 影響えいきょう（Impact）」で評価ひょうかされます。

  • リスク特定りすくとくてい：資産しさん・脅威きょうい・脆弱性ぜいじゃくせいを洗い出す
  • 定性的分析ていせいてきぶんせき（Qualitative）：主観的しゅかんてきな評価ひょうか、高中低でランク付け
  • 定量的分析ていりょうてきぶんせき（Quantitative）：ALE = SLE × ARO の数式で算出
  • リスク対応りすくたいおう：回避・軽減けいげん・転移てんい・受容じゅよう
  • 残留リスクざんりゅうりすく（Residual Risk）：対策後たいさくごに残るリスク
KEY TERMS
ALE SLE × ARO 脆弱性ぜいじゃくせい 脅威きょうい 残留リスクざんりゅうりすく
GRC
ガバナンスがばなんす・コンプライアンスこんぷらいあんす・規制きせい

組織そしきは法令ほうれい・規制きせい・業界標準ぎょうかいひょうじゅんに従いセキュリティ方針ほうしんを策定する必要があります。

  • GDPR：EU一般データ保護規則いっぱんでーたほごきそく
  • HIPAA：米国医療情報いりょうじょうほう保護法
  • PCI DSS：決済けっさいカード情報じょうほうセキュリティ基準きじゅん
  • SOX法：財務情報ざいむじょうほうの正確性せいかくせい確保
  • ISMSフレームワーク：ISO/IEC 27001
KEY TERMS
GDPRHIPAA ISO 27001 コンプライアンスこんぷらいあんす
BCP
BCP / DRP：事業継続じぎょうけいぞくと復旧計画ふっきゅうけいかく

BCP（事業継続計画じぎょうけいぞくけいかく）はシステム停止ていし時にも事業じぎょうを維持する戦略で、DRP（災害復旧計画さいがいふっきゅうけいかく）はITシステムしすてむの復旧ふっきゅう手順を定めます。

  • 目標復旧時間もくひょうふっきゅうじかん（RTO）：復旧ふっきゅうに許容きょようされる最大時間
  • 目標復旧時点もくひょうふっきゅうじてん（RPO）：許容きょようできるデータ損失量そんしつりょう
  • 業務影響分析ぎょうむえいきょうぶんせき（BIA）：重要業務じゅうようぎょうむとその影響を特定
  • MTD / MTTR / MTBF：各可用性かようせい指標
  • 代替サイトだいたいさいと：Hot / Warm / Cold Site の区別
ETH
ISC²職業倫理しょくぎょうりんり規範

ISC²の倫理綱領りんりこうりょうは全CISSP保持者が遵守すべき最高規範きはんであり、いかなる地域法ちいきほうよりも上位に位置します。

  • 社会しゃかい・公益こうえきを守る（Protect society, the common good）
  • 誠実せいじつかつ勤勉きんべんに行動こうどうする（Act honorably, honestly, justly）
  • 専門家せんもんかとして提供ていきょうする（Provide diligent and competent service）
  • 職業しょくぎょうを発展はってんさせる（Advance and protect the profession）
POL
セキュリティポリシーせきゅりてぃぽりしーの階層かいそう構造

組織そしきのセキュリティ文書体系ぶんしょたいけいは階層構造を持ちます。

  • 方針ほうしん（Policy）：最上位さいじょういの意思決定いしけってい文書
  • 基準きじゅん（Standard）：守るべき具体的ぐたいてきな仕様しよう
  • 手順書てじゅんしょ（Procedure）：実施じっしの具体的手順ぐたいてきてじゅん
  • ガイドラインがいどらいん（Guideline）：推奨すいしょうであり強制きょうせいでない

02
Asset Security

資産しさんセキュリティ

10% — 試験出題比率

情報資産じょうほうしさんの分類ぶんるい・所有権しょゆうけん・保護ほご・廃棄に至るライフサイクル全体を管理します。データの機密レベルきみつれべるに応じた取り扱い手順とりあつかいてじゅんと、プライバシーぷらいばしー規制への適合てきごうが重要テーマです。

CLS
データ分類でーたぶんるいと取り扱い手順とりあつかいてじゅん

政府せいふ系：最高機密さいこうきみつ（Top Secret）→ 機密きみつ（Secret）→ 部外秘ぶがいひ（Confidential）→ 公開こうかい（Unclassified）

  • 民間企業みんかんきぎょう：Confidential / Private / Sensitive / Public
  • データオーナーでーたおーなー：分類責任ぶんるいせきにんを持つ経営層けいえいそう
  • データカストディアンでーたかすとでぃあん：IT部門が技術的保護ぎじゅつてきほごを実施
  • データユーザーでーたゆーざー：方針ほうしんに従いデータを利用
DRM
データライフサイクルでーたらいふさいくると廃棄はいき

作成さくせい→保管ほかん→利用りよう→共有きょうゆう→アーカイブあーかいぶ→廃棄はいきの各フェーズで適切なコントロールが必要です。

  • 消去しょうきょ（Clearing）：上書きうわがきによる復元ふくげん困難化
  • パージぱーじ（Purging）：より徹底的てっていてきな抹消まっしょう（消磁・複数回上書き）
  • 物理破壊ぶつりはかい（Destruction）：シュレッダー・溶解・粉砕
  • データ残留でーたざんりゅう（Data Remanence）：削除後さくじょごも残る痕跡こんせき
PRI
プライバシーぷらいばしー保護と個人情報こじんじょうほう
  • PII（個人識別情報こじんしきべつじょうほう）：氏名・住所・マイナンバー等
  • PHI（医療健康情報いりょうけんこうじょうほう）：HIPAA対象たいしょう
  • 仮名化かめいか（Pseudonymization）：識別子を代替だいたい値に置換
  • 匿名化とくめいか（Anonymization）：再識別さいしきべつ不可能なレベルに除去
  • データ最小化さいしょうか（Data Minimization）：必要最低限のみ収集

03
Security Architecture and Engineering

セキュリティアーキテクチャあーきてくちゃとエンジニアリングえんじにありんぐ

13% — 試験出題比率

セキュリティを設計せっけいに組み込む原則（セキュリティバイデザインせきゅりてぃばいでざいん）と、暗号化あんごうか技術の理論・応用を学びます。OSIモデル、仮想化かそうか、クラウドセキュリティ、組込みシステムくみこみしすてむも対象たいしょうです。

CRY
暗号化あんごうかの基礎きそとアルゴリズムあるごりずむ

対称鍵暗号たいしょうかぎあんごう（Symmetric）：同じ鍵で暗号化あんごうか・復号ふくごう。高速だが鍵配送かぎはいそうが課題。

  • AES（128/192/256bit）：現在の標準ひょうじゅん
  • 3DES：DESの後継こうけい、段階的廃止だんかいてきはいし中
  • RSA：非対称鍵ひたいしょうかぎ、公開鍵こうかいかぎ・秘密鍵ひみつかぎペア
  • ECC（楕円曲線暗号だえんきょくせんあんごう）：短い鍵長で高安全性あんぜんせい
  • SHA-256 / SHA-3：ハッシュ関数はっしゅかんすう
KEY TERMS
AESRSA PKI 楕円曲線だえんきょくせん
SEC
セキュリティ設計原則せっけいげんそく
  • 最小権限さいしょうけんげん（Least Privilege）：必要最小限の権限のみ付与
  • 職務分掌しょくむぶんしょう（Separation of Duties）：単独たんどくで不正ふせいができない構造
  • 多層防御たそうぼうぎょ（Defense in Depth）：複数レイヤーの防御ぼうぎょ
  • フェイルセーフ（Fail-Safe）：障害時しょうがいじは安全側あんぜんがわへ
  • 完全性確認かんぜんせいかくにん（Complete Mediation）：全アクセスあくせすをチェック
  • ゼロトラスト（Zero Trust）：信頼しんらいしない、常に検証けんしょう
PKI
PKI とデジタル証明書でじたるしょうめいしょ

PKI（公開鍵基盤こうかいかぎきばん）はデジタル証明書でじたるしょうめいしょを通じて公開鍵こうかいかぎの真正性しんせいせいを保証するインフラです。

  • CA（認証局にんしょうきょく）：証明書しょうめいしょを発行はっこう・管理かんり
  • RA（登録局とうろくきょく）：申請者しんせいしゃの身元確認みもとかくにん
  • CRL / OCSP：失効しっこうした証明書しょうめいしょの確認
  • X.509：標準的ひょうじゅんてきな証明書しょうめいしょフォーマットふぉーまっと
CLD
クラウドと仮想化かそうかのセキュリティ
  • IaaS / PaaS / SaaS：責任共有せきにんきょうゆうモデル
  • ハイパーバイザーはいぱーばいざー（Hypervisor）：VM管理基盤かんりきばん
  • VM Escape：仮想機械かそうきかいを脱出だっしゅつするリスク
  • CASB：クラウドアクセスセキュリティブローカー
  • FaaS / コンテナ：新たな攻撃面こうげきめん

Communication and Network Security

通信つうしんとネットワークセキュリティ

13% — 試験出題比率

ネットワーク設計せっけい、伝送でんそうメディア、通信プロトコルつうしんぷろとこるのセキュリティを学びます。OSI参照モデルの各層、ファイアウォール、VPN、無線LANセキュリティが核心的な出題範囲しゅつだいはんいです。

OSI
OSI参照モデルさんしょうもでるとTCP/IP
  • L7 応用層おうようそう：HTTP, FTP, DNS, SMTP
  • L6 表現層ひょうげんそう：暗号化あんごうか・文字コード変換
  • L5 セッション層せっしょんそう：接続管理せつぞくかんり（TLSハンドシェイクはんどしぇいく）
  • L4 トランスポート層とらんすぽーとそう：TCP / UDP
  • L3 ネットワーク層ねっとわーくそう：IP, IPSec
  • L2 データリンク層でーたりんくそう：MAC, ARP, VLANs
  • L1 物理層ぶつりそう：ケーブル、無線
FW
ファイアウォールと境界防御きょうかいぼうぎょ
  • パケットフィルタリングぱけっとふぃるたりんぐ：L3/L4の静的せいてきルール
  • SPI（ステートフルインスペクションすてーとふるいんすぺくしょん）：接続状態せつぞくじょうたいを追跡
  • NGF（次世代ファイアウォール）：アプリ認識・IPS統合
  • WAF（ウェブアプリケーションファイアウォールうぇぶあぷりけーしょんふぁいあうぉーる）：L7で検査けんさ
  • DMZ（非武装地帯ひぶそうちたい）：公開サーバーこうかいさーばーの設置領域
VPN
VPN と暗号化通信あんごうかつうしんプロトコル
  • IPSec（トンネルモードとんねるもーど/トランスポートモード）
  • TLS 1.3：最新の安全あんぜんな伝送でんそうプロトコル
  • SSL VPN：HTTPSを使ったリモートアクセス
  • MPLS VPN：専用回線せんようかいせん代替
  • SD-WAN：ソフトウェア定義ネットワーク
WLS
無線LANむせんらんセキュリティ
  • WEP：脆弱ぜいじゃく、使用禁止きんし
  • WPA2（AES/CCMP）：現行標準ひょうじゅん
  • WPA3：最新、SAE認証にんしょう採用
  • 802.1X/EAP：企業向けきぎょうむけ認証にんしょうフレームワーク
  • 不正AP（Rogue AP）・Evil Twin：偽APにせえーぴー攻撃

05
Identity and Access Management

識別しきべつ・認証にんしょう・認可にんか（IAM）

13% — 試験出題比率

「誰が」「何に」「どの範囲で」アクセスできるかを管理します。識別しきべつ（Identification）・認証にんしょう（Authentication）・認可にんか（Authorization）・説明責任せつめいせきにん（Accountability）の4ステップが基本です。

MFA
多要素認証たようそにんしょう（MFA）
  • 知識（Something you know）：パスワード・PIN
  • 所持しょじ（Something you have）：スマートカード・OTPトークン
  • 生体せいたい（Something you are）：指紋・顔認証・虹彩
  • 場所ばしょ（Somewhere you are）：地理的位置情報
  • FRR（本人拒否率ほんにんきょひりつ）/ FAR（他人許容率たにんきょようりつ）/ CER（等価誤り率とうかあやまりりつ）
ACC
アクセス制御あくせすせいぎょモデル
  • DAC（任意にんいアクセス制御）：所有者しょゆうしゃが権限を設定
  • MAC（強制きょうせいアクセス制御）：ラベルと機密区分きみつくぶんで制御
  • RBAC（役割やくわりベース）：役職やくしょくに権限を付与
  • ABAC（属性ぞくせいベース）：多様な属性で動的制御
  • Bell-LaPadula：機密性きみつせい重視モデル（No Write Down）
  • Biba：完全性かんぜんせい重視モデル（No Write Up）
SSO
SSO・フェデレーション・特権管理とっけんかんり
  • SAML 2.0：XMLベースの認証にんしょうフェデレーション
  • OAuth 2.0：認可にんか委譲フレームワーク
  • OpenID Connect：OAuth上の認証にんしょうレイヤー
  • PAM（特権アクセス管理とっけんあくせすかんり）：管理者権限の監視
  • ゼロトラスト IAM：継続的けいぞくてきな検証けんしょう

06
Security Assessment and Testing

セキュリティ評価ひょうかとテスト

12% — 試験出題比率

脆弱性ぜいじゃくせい評価ひょうか、侵入テストしんにゅうてすと、監査かんさ、ログ監視かんしを通じてセキュリティ態勢たいせいを継続的に検証けんしょうします。SIEM、脆弱性スキャンぜいじゃくせいすきゃん、ペネトレーションテストの違いを理解することが重要です。

PEN
侵入テストしんにゅうてすと（ペンテスト）
  • ブラックボックス：事前情報じぜんじょうほうなし、外部攻撃者こうげきしゃ視点
  • ホワイトボックス：全情報じょうほう開示、内部ないぶ視点
  • グレーボックス：部分的な情報じょうほうあり
  • 偵察ていさつ→スキャン→侵入しんにゅう→維持いじ→痕跡消去こんせきしょうきょ
  • SOW（作業範囲定義書さぎょうはんいていぎしょ）必須
AUD
監査かんさとコンプライアンステストてすと
  • 内部監査ないぶかんさ：自社じしゃチームによる定期検査ていきけんさ
  • 外部監査がいぶかんさ：第三者だいさんしゃの独立検証どくりつけんしょう
  • SOC 1 / SOC 2 / SOC 3：報告書ほうこくしょの種類
  • SIEM：ログ集約しゅうやく・相関分析・警告けいこく
  • IDS/IPS：侵入検知しんにゅうけんち・侵入防止しんにゅうぼうし
VUL
脆弱性管理ぜいじゃくせいかんりプログラム
  • CVSS：共通脆弱性評価きょうつうぜいじゃくせいひょうかシステム（0-10スコア）
  • CVE：脆弱性ぜいじゃくせいの共通識別子きょうつうしきべつし
  • パッチ管理ぱっちかんり：定期的ていきてきな適用てきようとテスト
  • Zero-day：未知みちの脆弱性ぜいじゃくせい（パッチなし）

07
Security Operations

セキュリティ運用うんよう

13% — 試験出題比率

日常的にちじょうてきなセキュリティ運用うんようの管理かんり：インシデント対応いんしでんたいおう、調査ちょうさ・フォレンジクスふぉれんじくす、物理セキュリティ、変更管理へんこうかんりを扱います。SOCとNOCの役割やくわりの違いも重要です。

IR
インシデント対応いんしでんたいおうの6フェーズ
  • ① 準備じゅんび（Preparation）：計画・訓練・ツール整備
  • ② 検知けんちと分析ぶんせき（Detection & Analysis）
  • ③ 封じ込めふうじこめ（Containment）：短期・長期戦略
  • ④ 根絶こんぜつ（Eradication）：マルウェア・バックドアばっくどあ除去
  • ⑤ 復旧ふっきゅう（Recovery）：システム再稼働さいかどう
  • ⑥ 事後検討じごけんとう（Lessons Learned）：PIR報告書ほうこくしょ
FOR
デジタルフォレンジクスでじたるふぉれんじくす
  • 証拠保全しょうこほぜん（Evidence Preservation）：改ざんかいざんなし
  • Chain of Custody（証拠連鎖しょうこれんさ）：取り扱い記録とりあつかいきろく
  • 揮発性順序きはつせいじゅんじょ：CPU→RAM→SWAP→HDD→バックアップ
  • Locard交換原則こうかんげんそく：接触せっしょくは必ず痕跡を残す
CHG
変更管理へんこうかんりと構成管理こうせいかんり
  • RFC（変更要求へんこうようきゅう）→ CAB承認しょうにん→実装じっそう→検証けんしょう
  • CMDB（構成管理データベースこうせいかんりでーたべーす）
  • バックアウト計画ばっくあうとけいかく：変更失敗時のロールバックろーるばっく
  • 緊急変更きんきゅうへんこう（Emergency Change）：事後承認じごしょうにんあり
PHY

Software Development Security

ソフトウェア開発かいはつセキュリティ

11% — 試験出題比率

セキュリティをSDLC（ソフトウェア開発そふとうぇあかいはつライフサイクル）の初期段階から組み込む「シフトレフトしふとれふと」の考え方を学びます。一般的な脆弱性ぜいじゃくせい、安全なコーディングあんぜんなこーでぃんぐ手法、API、サードパーティ管理かんりが含まれます。

SDL
SDLC と安全あんぜんな開発手法
  • 要件定義ようけんていぎ→設計→実装じっそう→テスト→展開てんかい→廃棄はいき
  • DevSecOps：CI/CDパイプラインにセキュリティ統合
  • 脅威モデリングきょういもでりんぐ（STRIDE）：設計段階で実施
  • SAST：静的解析せいてきかいせき（ソースコード検査けんさ）
  • DAST：動的解析どうてきかいせき（実行中テスト）
OWA
OWASP Top 10 と主要脆弱性ぜいじゃくせい
  • SQLインジェクション：DB不正操作ふせいそうさ
  • XSS（クロスサイトスクリプティングくろすさいとすくりぷてぃんぐ）：スクリプト注入
  • CSRF：他サイトたさいとを騙したリクエストりくえすと偽造
  • IDOR：安全でない直接オブジェクト参照あんぜんでないちょくせつおぶじぇくとさんしょう
  • バッファオーバーフロー：メモリ境界めもりきょうかい超過
API
APIセキュリティとサプライチェーン
  • REST API：JWT・OAuth2.0による認証にんしょう
  • 入力値検証にゅうりょくちけんしょう：全パラメータをホワイトリスト検証けんしょう
  • SBOM（ソフトウェア部品表そふとうぇあぶひんひょう）：依存関係いぞんかんけいの可視化
  • オープンソース依存関係いぞんかんけいの脆弱性ぜいじゃくせい監視
  • SolarWinds型攻撃こうげき：サプライチェーンリスク
