01 Metric & Gauge Visualizations
ClassicLens
✓ DONE ▾

A Metric visualization displays a single aggregated number (count, sum, average, etc.). A Gauge shows that value relative to a min/max range, like a speedometer. Both are useful for KPI dashboards.

🎯
Exam objective: "Create a Metric or Gauge visualization that displays a value satisfying a given criteria." You'll often be asked to show a Count, Sum, or Unique Count of a field — sometimes filtered by a specific condition.
How to create (Lens)
1
Open Visualize Library → Create visualization
Choose Lens as the editor.
2
Select chart type: Metric
Click the chart type icon (top-left in Lens) → select Metric.
3
Choose your data view & metric
Drag a field to the Primary metric slot, or choose an aggregation like Count. For Gauge, also set Minimum and Maximum.
4
Apply KQL filter if needed
Use the Filters bar or add a filter in the aggregation. e.g., only count documents where status:error.
5
Save with the exact task name
Click Save → enter the name from the task → Save to library.

04 Sub-bucket Aggregations (Split Series)
LensClassic
✓ DONE ▾

A sub-bucket aggregation splits a visualization into multiple series or segments by a second dimension. In Lens, this is called "Break down by". In Classic, it's "Add sub-buckets" → Split Series.

Aggregation Structure
AGGREGATION CONFIGURATION
METRIC Count of documents
BUCKET X-axis: Date Histogram on @timestamp (interval: 1h)
SUB-BUCKET Split Series: Terms on http.response.status_code (top 5)

The example above creates a line chart with one line per HTTP status code, all over time — a classic multi-series visualization.

Sub-bucket types
TypeUse case
TermsSplit by keyword field values (e.g., per country, per host)
Date HistogramSplit by time intervals (can be nested)
RangeSplit numeric field into custom ranges
IPv4 RangeSplit by IP address ranges
Significant TermsTerms that are unusually frequent
FiltersSplit by multiple KQL filter conditions
💡
In Lens, "Break down by" achieves the same result as sub-buckets. You can add it by dragging a field to the "Break down by" slot in the layer panel.
05 Moving Average, Derivative & Serial Diff
LensTSVB
✓ DONE ▾

These are pipeline aggregations — they run on the output of another aggregation (not raw documents). They're used to smooth data, detect rate-of-change, or compare consecutive buckets.

Three Pipeline Aggregations
AggregationWhat it doesUse case
Moving AverageRolling average of the last N bucketsSmoothing out noise in time series
DerivativeRate of change between consecutive buckets (bucket[n] − bucket[n-1])Detecting spikes/drops in velocity
Serial DifferencingDifference between current and a bucket N periods agoRemoving seasonal patterns, finding anomalies
# How to add a Moving Average in Lens 1. Create a Line chart with Date Histogram (X) + Count (Y) 2. Click "+" to add a second layer, OR click the Y-axis metric 3. Select "Moving Average" from the formula functions Formula example: moving_average(count(), window=5) 4. Style the moving average line differently (dashed, different color) # In Classic editor (Aggregation-based) Metrics → Add metric → Moving Avg Parent pipeline: select your existing Count metric Window: 5 (number of buckets to average)
Visual Demo — Moving Average Smoothing

07 Elastic Maps (Geo Data)
Maps
✓ DONE ▾

Elastic Maps is Kibana's dedicated mapping application. It visualizes geo_point and geo_shape fields from your Elasticsearch indices as interactive map layers.

🎯
Exam objective: "Using geo data, create an Elastic map that satisfies a given criteria." You need to know how to add layers, choose layer types, and configure them.
Map Layer Types
📍
Document Points
Individual geo_point documents as dots on the map
🔥
Heat Map
Density of geo_points as a heat gradient
🗺️
Choropleth
Color regions (countries, states) by a metric value
⬡
Clusters & Grids
Aggregate many points into grid cells or clusters
🧩
EMS Boundaries
Elastic Maps Service world/regional boundaries
🔷
geo_shape Layer
Polygons, lines, complex geographic shapes
Creating an Elastic Map
# Navigation: Kibana → Maps → Create map Step 1: Click "Add layer" Step 2: Choose layer source: — Documents (individual geo_point records) — Clusters and grids (aggregated — faster for large data) — Heat map — EMS boundaries (for choropleth with Elasticsearch data) Step 3: Select your data view and geo field Step 4: Configure layer settings: — Layer style (color, size, opacity) — Apply KQL filter to this layer — Set tooltip fields Step 5: Repeat for multiple layers Step 6: Set map title and save
⚠️
The geo field must be mapped as geo_point or geo_shape in Elasticsearch. If the field isn't appearing in the layer config, check the mapping — string fields won't work as geo fields.
08 TSVB — Time Series Visual Builder
TSVB
✓ DONE ▾

TSVB (Time Series Visual Builder) is a powerful visualization tool for time-series data. It supports advanced features like filter ratios, moving averages, and mathematical formulas between series — things that aren't easily done in Lens.

🎯
High-priority exam topic! Multiple objectives relate to TSVB. Know how to create Time Series, Metric, Gauge, Table, and Top N panel types within TSVB.
TSVB Panel Types
Panel TypeWhat it shows
Time SeriesLine/bar/area charts over time. Supports multiple series, custom intervals.
MetricSingle value (last value or aggregation). Supports threshold-based color changes.
Top NHorizontal bar chart of top N values for a terms field.
GaugeGauge/dial showing a current value against a range.
TableTabular view of aggregated values, grouped by a terms field.
MarkdownRich-text panel with optional embedded Mustache variables.
Key TSVB Features
# How to create a TSVB Time Series visualization 1. Kibana → Visualize → Create → TSVB 2. The default panel type is Time Series 3. Under Data tab → configure Series: — Aggregation: Count / Sum / Avg / Max / Min… — Field: choose numeric field if needed — Group by: optionally split by a terms field 4. Click "+" to add a second series (multi-series chart) 5. Under Panel options: set index pattern, time field, interval 6. Switch panel type (top-left) to Metric / Top N / Gauge / Table # Filter Ratio (TSVB exclusive) Series aggregation → Filter Ratio Numerator filter: status.keyword : "error" Denominator filter: * (all docs) → Shows percentage of errors to total requests
Multiple Series on One TSVB Chart
# Add multiple series in TSVB Time Series Series 1: Count (all requests) → label: "Total" → color: blue Series 2: Count + Filter: status:200 → label: "Success" → color: green Series 3: Count + Filter: status>=400 → label: "Errors" → color: red Each series can have its own: — Aggregation type — KQL filter — Color & line style — Sub-aggregation (optional)

09 TSVB: Filter Ratio, Moving Average & Math
TSVB
✓ DONE ▾

TSVB provides three advanced series types that go beyond basic aggregations — perfect for calculating derived metrics.

FeatureHow it worksExample
Filter RatioDivides the count of docs matching filter A by docs matching filter BError rate = errors / total requests
Moving AverageRolling mean over N previous time bucketsSmoothed request count over 5 intervals
MathArbitrary formula combining other series using variablesparams.a * 100 / params.b
Positive RateLike derivative but always positive; for monotonic countersBytes received per second from cumulative total
Calculation (bucket script)Scripts on existing metrics in a bucketCustom percentage calculation
# Math aggregation example in TSVB # Scenario: show error rate as a percentage Series A (variable: errors): Aggregation: Count Filter: http.response.status_code >= 500 Series B (variable: total): Aggregation: Count Filter: * Series C: Math Formula: params.errors / params.total * 100 → Displays error percentage over time
10 TSVB: Metric, Gauge, Table & Top N
TSVB
✓ DONE ▾

Within TSVB, you can switch the panel type (top-left dropdown) to show data in different ways. All use the same data config but render it differently.

# Switching TSVB panel types Navigate to: Visualize → Create → TSVB → [Panel Type dropdown, top-left] METRIC: Shows the last value of the series as a large number Set color thresholds: green/yellow/red based on value ranges e.g. green if < 100ms, yellow if < 300ms, red if >= 300ms TOP N: Horizontal bar chart showing top N values of a terms field Configure: Group by → Terms → field → size (e.g. 5) Useful for "top 5 slowest endpoints" TABLE: Similar to Top N but in tabular format Can show multiple metrics per row GAUGE: Dial-style visualization Set min/max values and color bands
💡
In TSVB Metric, you can set color rules under Panel Options → Color Rules. Add multiple rules to get green/yellow/red traffic-light behavior based on thresholds.
11 Tag Cloud Visualization
Classic
✓ DONE ▾

A Tag Cloud displays terms from a keyword field with font size proportional to their frequency (or another metric). Only available in the Classic (Aggregation-based) editor, not in Lens.

🎯
Exam objective: "Create a Tag Cloud visualization on a keyword field of an index." Make sure to use a .keyword field (not a text field) as the Tags bucket.
Creating a Tag Cloud
1
Visualize → Create → Tag Cloud
Only available under "Aggregation based" visualizations.
2
Select data view
Choose the appropriate index/data view.
3
Buckets → Tags → Terms
Field: response.keyword or any .keyword field. Set size (e.g., 10).
4
Metric (optional)
Default is Count. Can change to Sum/Avg of a numeric field — the metric controls font size.
5
Save with the exact name from the task

14 Creating a Dashboard
Dashboard
✓ DONE ▾

A Dashboard is a collection of visualizations, saved searches, maps, and controls assembled on a single canvas. The exam will ask you to create one combining several of the above.

Full Dashboard Workflow
1
Navigate to Dashboard → Create dashboard
2
Add panels from library
Click "Add from library" → select your saved visualizations, searches, maps.
3
Create inline visualizations
Click "Create visualization" to build directly from the dashboard — saves to library automatically.
4
Resize and arrange panels
Drag corners to resize. Drag the panel header to reposition. Click the panel menu (⋮) for options.
5
Set the global time range
Time picker (top-right) applies to ALL panels simultaneously.
6
Save with the exact task name
Click "Save" → enter the name specified in the task. Double-check spelling.
🎯
Exam tips: Dashboard tasks often say "create a dashboard that consists of [list of specific visualizations]." Make sure every listed panel is present. After saving, verify by opening the dashboard from the Dashboard list.
Dashboard Controls & Filters
FeatureBehavior
Global time filterApplies to all time-aware panels. Override per-panel if needed.
Dashboard-level KQLTyped in the search bar; applies to ALL panels simultaneously.
Dashboard-level filtersFilter chips at top; apply to all panels. Can be pinned.
Panel-level filterRight-click chart element → "Filter for value" → only that panel's query changes.
Drill-downConfigure in panel settings → "Create drilldown" to navigate to another dashboard on click.
