01 Define an Index Pattern / Data View ✓ DONE ▾
Concept

A Data View (formerly called "Index Pattern") tells Kibana which Elasticsearch indices to work with. It also lets you designate a timestamp field, which enables time-based filtering across all of Kibana. You must create a Data View before you can search or visualize any data.

🎯
Exam relevance: Almost every task begins with "In the Default space, create a Data View from the filebeat-* index pattern with the @timestamp field set as the time filter field." Know this cold.
Step-by-Step
# How to create a Data View in Kibana 1. Open Kibana → Stack Management → Data Views 2. Click "Create data view" 3. Enter the index pattern name (e.g. filebeat-*) — Use * as a wildcard to match multiple indices 4. Select a Timestamp field (usually @timestamp) — OR choose "I don't want to use the time filter" 5. Click "Save data view to Kibana"
⚠️
If no timestamp field exists in the index, you must select "I don't want to use the time filter." The exam may ask you to do this explicitly — read the task carefully.
💡
You can also create Data Views directly from Discover by clicking the data view name dropdown and selecting "Create a data view."
Key Concepts
TermMeaning
Index Pattern / Data ViewA saved configuration pointing to one or more Elasticsearch indices. e.g. logs-*, metricbeat-7.*
Wildcard (*)Matches any index. filebeat-* matches filebeat-8.0, filebeat-8.1, etc.
Time Filter FieldThe date field used for time-based filtering. Typically @timestamp.
Default SpaceThe default Kibana Space. Exam tasks often specify "in the Default space."
02 Set the Time Filter ✓ DONE ▾
Concept

The Time Filter (top-right in Kibana) restricts all queries to a date/time range. It only works when the Data View has a timestamp field. You can set it via a quick select, a relative range, or an absolute range.

🎯
Exam tip: Tasks will say things like "filter to the last 7 days" or "show data from January 1, 2023 to December 31, 2023." Use the Absolute tab when given specific dates.
Time Filter Options
ModeHow to useExample
Quick selectPre-built options for common rangesLast 15 minutes, Last 7 days, Today
RelativeRelative to nownow-7d, now-1h/h
AbsoluteExact dates (calendar picker)2023-01-01 → 2023-12-31
NowContinuously refresheslast 1 hour, auto-refresh every 30s
# Relative time expressions now-1d # 1 day ago from now now-7d/d # 7 days ago, rounded to start of day now/M # start of current month now-1M/M # start of last month now-1y # 1 year ago
💡
You can also set the time filter directly in the search bar using KQL's date expressions, or by brushing over a time histogram in Discover.
03 KQL — Kibana Query Language ✓ DONE ▾
Concept

KQL (Kibana Query Language) is a simplified query syntax used in Kibana's search bar to filter documents. It's more readable than raw Elasticsearch DSL and the default query language in modern Kibana.

KQL Syntax Reference
field : value
Exact match on a field
status : 200
field : "phrase here"
Phrase match (preserve word order)
message : "failed login"
field : val*
Wildcard — * matches any chars
url.path : /api/*
AND / OR / NOT
Boolean operators (uppercase)
status:200 AND NOT method:GET
field >= value
Range comparison: >, >=, <, <=
response.bytes >= 10000
field : (a OR b)
Multiple values for one field
status : (400 OR 500)
field : *
Field exists (not null)
user.email : *
nested.field : value
Dot notation for nested fields
system.auth.ssh.event : failed

⚠️

🎯
Exam tip: When the task says "display only documents where field X has value Y," write a KQL query. Don't forget to save the search with the name specified in the task.

04 Create and Pin Filters ✓ DONE ▾
Concept

Filters in Kibana are persistent, toggleable query clauses that supplement the search bar. They appear as chips below the search bar and can be added via the UI (clicking on field values in Discover) or manually via the "Add filter" dialog. Pinned filters persist across all apps in Kibana until unpinned.

Ways to Add a Filter
## Method 1: Click a field value in Discover → Expand a document → click "+" next to a field value This adds a filter: field IS value ## Method 2: Add filter dialog (manual) → Click "+ Add filter" (below search bar) → Choose field, operator, value → Save ## Method 3: From a visualization → Click a bar/segment on a chart → "Filter for value"
Filter Actions
ActionHow
Toggle on/offClick the filter chip to enable/disable without removing
Invert (negate)Click filter menu → "Invert filter" — adds NOT to the condition
Pin (persist globally)Click filter menu → "Pin across all apps" — stays active when navigating
EditClick filter menu → "Edit filter" — modify field/operator/value
RemoveClick ✕ on the filter chip or "Delete" from menu
🎯
Exam tip: If asked to "create and pin a filter," make sure you both create it AND click "Pin across all apps" from the filter's dropdown menu.

05 Apply Search Criteria to Visualizations & Dashboards ✓ DONE ▾
Concept

Search criteria (KQL + filters + time range) from Discover can be saved and applied to visualizations or included in dashboards. This is key to the exam workflow: search → filter → save → use in dashboard.

Workflow
# Full workflow for exam tasks Step 1: Create your KQL search in Discover e.g. system.auth.ssh.event : failed Step 2: Add any needed filters (or pin them) Step 3: Set the correct time range Step 4: Save the search via Save → Saved search (use the exact name from the task!) Step 5: Create a visualization using the same data view — The saved search will auto-apply the index — Add your own KQL/filters in the viz if needed Step 6: Add to dashboard — Click "Add from library" or create inline — Dashboard inherits the time filter from its own time range
💡
When you add a saved search panel to a dashboard, the search's embedded KQL/filters and the dashboard's global time filter both apply simultaneously — their effects are combined (AND logic).
🎯
Naming matters! Exam tasks specify exact names for saved searches, visualizations, and dashboards. Points are awarded for correct naming. Copy names character-by-character from the task.
